## Usage

These rules are automatically applied when working in the project. They provide:

- **Consistent patterns** across all components
- **Type safety** with proper TypeScript usage
- **Performance optimization** guidelines
- **Security best practices**
- **Code quality standards**

## Key Principles

1. **Consistency:** Follow established patterns throughout the project
2. **Type Safety:** Use TypeScript strictly with proper type definitions
3. **Performance:** Optimize for bundle size and runtime performance
4. **Maintainability:** Write code that's easy to understand and modify
5. **Security:** Follow security best practices
6. **Testing:** Maintain high test coverage with proper testing patterns

## Contributing

When adding new components or features:

1. Follow the patterns established in these rule files
2. Use the existing component structure as a template
3. Ensure all TypeScript types are properly defined
4. Include proper tests and documentation
5. Follow the established naming conventions

## Questions?

If you have questions about any of these rules or need clarification on specific patterns, refer to the existing codebase examples or ask for guidance.
